# AI-Native CRM

Multi-tenant CRM platform with a React frontend, Spring Boot backend, and Python AI service.

The current product surface includes:
- core CRM for leads, contacts, companies, deals, tasks, calendar, email, documents, products, quotes, and invoices
- revenue operations, forecasting, reporting, governance, and workflow automation
- campaigns, nurture journeys, and marketing attribution
- support cases, SLA automation, service dashboards, and field service work orders
- contracts, CPQ guardrails, and customer data governance
- workspace integrations for Microsoft 365, Google Workspace, QuickBooks, Xero, and more

## Architecture

- Frontend: React + TypeScript + Vite
- Backend: Spring Boot + PostgreSQL + Redis + RabbitMQ
- AI Service: FastAPI for AI chat, custom AI reports, forecasting, and scoring
- Runtime: Docker Compose for backend infrastructure and services

## Recommended Local Run Flow

Use Docker Compose for the backend stack, then run the frontend locally.

### Prerequisites

- Node.js 18+
- npm 10+
- Docker Desktop / Docker Engine with Compose support
- Python 3.11+ if you want to run the AI service outside Docker
- Java 17+ and Maven if you want to run backend tests outside Docker

## Quick Start

### 1. Install frontend dependencies

From the repo root:

```bash
npm install
```

### 2. Configure backend environment

Copy the backend env template and fill in the values you need:

```bash
cd backend
copy .env.example .env
```

Important variables:
- `GROQ_API_KEY`
- `OPENAI_API_KEY`
- `JWT_SECRET`
- `MAIL_HOST`
- `MAIL_PORT`
- `MAIL_USERNAME`
- `MAIL_PASSWORD`
- `TENANCY_DATABASE_CREDENTIALS_KEY`

### 3. Start backend + database + AI service

From [`backend/README.md`](/C:/Users/cni.alad/Documents/Projects/Projects/Cicosy-CRM/ai-native-crm/backend/README.md):

```bash
docker compose -f docker-compose.run.yml up -d --build
```

This starts:
- PostgreSQL on `localhost:5433`
- Redis on `localhost:6379`
- RabbitMQ on `localhost:5672`
- RabbitMQ UI on `http://localhost:15672`
- Spring Boot backend on `http://localhost:8080`
- AI service on `http://localhost:8000`

### 4. Start the frontend

Back in the repo root:

```bash
npm run dev
```

Vite runs on:
- Frontend: `http://localhost:5173`

## Smoke Testing

From the repo root:

```bash
npm run smoke
```

Run backend-heavy smoke without AI assertions:

```bash
CRM_SMOKE_SKIP_AI=true npm run smoke
```

The smoke suite exercises the newer modules too, including:
- contracts and CPQ
- reporting and forecasting
- cases and service operations
- field service work orders
- integrations surfaces
- AI chat and scoring paths

## Reporting Architecture

- Standard deterministic CRM reports are generated by the Java backend.
- Custom AI reports are generated by the Python AI service.
- The frontend reporting workspace routes each request to the correct service.
